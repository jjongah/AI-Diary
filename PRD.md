# AI 대화형 일기 서비스 PRD

- 문서 버전: v0.4
- 작성일: 2026-07-20
- MVP 확정일: 2026-07-20
- 목표 개발 기간: 2일
- 앱 유형: 외부 계정 인증 없는 로컬 프로필 기반 반응형 웹 앱
- 문서 상태: MVP 구현 내용 반영본

## 1. 제품 한 줄 정의

반복되는 일상 속에서 지나치기 쉬운 작은 사건과 감정을 따뜻한 AI 서재 관리자와 대화로 꺼내고, 한 편의 일기로 정리해 다시 만날 수 있게 하는 웹 일기 서비스.

## 2. 배경과 문제

학교, 집, 아르바이트처럼 비슷한 하루가 반복되면 실제로 여러 일을 겪었더라도 시간이 지난 뒤에는 남은 기억이 거의 없다고 느낄 수 있다. 기존 일기 서비스는 빈 화면에서 사용자가 처음부터 글을 완성해야 해 피곤한 날일수록 기록을 시작하기 어렵다.

이 제품은 “특별한 일이 있어야 기록할 가치가 있다”는 부담을 줄이고, 대화를 통해 작은 기억을 발견하도록 돕는다. 핵심 가치는 생산성이나 자기계발이 아니라 다음 세 가지다.

1. 기록을 시작하기 쉽게 만들기
2. 평범한 하루에도 의미가 있었음을 느끼게 하기
3. 시간이 지난 뒤 당시의 기억을 쉽게 되찾게 하기

## 3. 목표와 비목표

### 3.1 MVP 목표

- 사용자가 AI와 3~10분 대화한 뒤 오늘의 일기 초안을 얻을 수 있다.
- 사용자가 대화 중 사진 또는 파일을 올리고, 지원 형식의 내용을 AI 초안에 반영할 수 있다.
- 사용자가 초안을 직접 수정하고 대화 중 올린 첨부물과 함께 저장할 수 있다.
- 사용자가 캘린더형과 목록형으로 지난 일기를 탐색할 수 있다.
- 사용자가 과거 날짜를 골라 일기를 추가하거나 수정할 수 있다.
- 사용자가 날짜와 내용에 관한 자연어 질문으로 지난 기록을 찾을 수 있다.
- 사용자가 `6월`, `29일`, `2026년 6월 29일` 같은 한국어 날짜 표현과 기분·날씨로 지난 기록을 찾을 수 있다.
- 사용자가 AI 서재 관리자의 대화 말투를 친근하게, 예의바르게, 정중하게 중에서 선택할 수 있다.
- 사용자가 이름과 선택 정보(나이, 성별, 생일)를 담은 로컬 프로필을 만들고 수정할 수 있다.
- AI 서재 관리자가 사용자의 프로필 이름을 대화 중 자연스럽게 호칭으로 사용할 수 있다.
- 모든 일기, 대화, 사진과 파일을 현재 브라우저에 로컬 저장한다.
- 사용자가 사진과 파일을 포함한 전체 데이터를 백업하고 복원할 수 있다.
- 외부 회원가입 없이 로컬 프로필 입장, 일기 CRUD, 로컬 파일 저장, AI 호출, 날씨 배경까지 시연 가능한 상태로 만든다.

### 3.2 MVP 비목표

- AI 심리 상담, 진단 또는 치료
- 음성 대화 및 음성 일기
- 여러 사람이 함께 쓰는 일기
- 고급 의미 기반 벡터 검색
- 사진 자체의 시각적 내용을 별도 태그로 저장하고 검색하는 기능
- 감정 통계, 주간·월간 리포트
- 세계수 성장 애니메이션과 복잡한 게임 시스템
- DOCX, HWP, ZIP 등 임의 파일 형식의 AI 내용 분석
- 네이티브 모바일 앱과 오프라인 동기화
- 여러 기기 간 자동 동기화

## 4. 핵심 사용자

### 1차 사용자

- 반복되는 학교·직장·아르바이트 생활로 하루를 기록할 에너지가 부족한 사람
- 글쓰기를 어려워하지만 대화에는 비교적 편안함을 느끼는 사람
- 시간이 흐른 뒤 자신의 일상과 감정을 다시 보고 싶은 사람

### 핵심 사용자 상황

“오늘도 별일 없었다고 생각했지만, 서재 관리자와 몇 마디 나누다 보니 점심에 친구가 건넨 말과 퇴근길의 냄새가 떠올랐다. AI가 정리한 글을 조금 고쳐 저장했고, 달력에 새 잎이 생긴 것을 보며 오늘도 사라지지 않았다고 느낀다.”

## 5. 제품 원칙

1. **평범함을 존중한다.** 사용자의 하루를 과장하거나 억지로 교훈화하지 않는다.
2. **사용자가 최종 저자다.** AI 결과는 항상 초안이며 저장 전 수정할 수 있다.
3. **대화는 자연스럽다.** 질문만 반복하지 않고 잡담, 공감, 의견과 필요한 질문을 맥락에 맞게 주고받는다.
4. **기록은 사용자의 것이다.** 삭제, 수정, AI 사용 범위를 사용자가 통제할 수 있어야 한다.
5. **따뜻하지만 인간인 척하지 않는다.** AI임을 명확히 하고 전문 상담가를 대체한다고 표현하지 않는다.

## 6. MVP 범위

### P0 — 반드시 구현

- 세계수 이미지가 있는 첫 화면
- 외부 계정 인증 없이 로컬 프로필 생성 후 `OO님의 서재 들어가기`
- 프로필 이름 필수 입력, 나이·성별·생일 선택 입력 및 설정에서 수정
- AI 대화에 프로필 이름과 선택 기본 정보 전달
- 메인 화면의 월간 달력과 오늘 날짜 표시
- 달력/목록 보기 전환
- 일기가 있는 날짜에 기분 아이콘, 저장된 날씨 아이콘과 제목 표시(날씨 모름은 아이콘 생략)
- 일기 생성, 조회, 수정, 삭제
- 과거 날짜 일기 추가
- AI 작성과 수동 작성 진입 경로
- IndexedDB 기반 일기·대화·설정·첨부물 로컬 저장
- 채팅 중 사진/파일 업로드, 미리보기 및 다시 열기
- 채팅, 편집 화면과 완성된 일기에서 사진을 파일명이 아닌 실제 이미지로 표시
- 지원 형식(JPG, PNG, WEBP, PDF, TXT)의 내용을 AI 일기 초안에 반영
- AI 대화 세션과 메시지 저장
- 완성된 일기의 `이 날의 대화`에서는 새 채팅을 열지 않고 당시 대화 원문을 읽기 전용으로 표시
- 대화를 바탕으로 일기 초안 생성
- AI 대화 말투 3종 선택 및 설정 저장
- 날짜·키워드·대화·기분·날씨를 대상으로 지난 기록 검색
- `29일`, `6월`, `6월 29일`, `2026년 6월`, `YYYY-MM-DD` 등 날짜 표현 지원
- 사진·파일을 포함하는 전체 ZIP 백업 및 복원
- Gemini API 키의 로컬 서버 환경 변수 보관
- 현재 날씨에 따른 맑음/흐림/비/눈 배경 레이어와 날씨 아이콘·온도
- 일기 저장 시 날씨 스냅샷과 기분 저장
- 과거 날짜 일기는 날씨 모름을 기본값으로 하고 사용자가 직접 지정 가능
- 모바일 반응형

### P1 — 시간이 남으면 구현

- 프로필 전환 또는 여러 로컬 프로필 지원
- 검색 결과 정렬 및 검색 조건 필터 UI
- 이미지 업로드 시 검색용 설명·태그 생성

### P2 — 후속 버전

- 세계수가 기록 수에 따라 성장하는 기능
- 주간·월간 회고 및 감정 흐름
- 벡터 기반 의미 검색
- 음성 입력
- 알림과 기록 습관 기능
- 개별 일기 내보내기(PDF/Markdown)
- 종단 간 암호화 또는 사용자 관리 암호화 키
- 첨부 이미지·문서의 자동 설명·태그 색인 및 내용 검색
- 암호로 보호된 백업 파일

## 7. 정보 구조와 화면

### 7.1 시작 화면

구성:

- 중앙 세계수 일러스트
- 서비스 이름과 한 문장 소개
- 프로필이 없을 때 `프로필 만들고 들어가기`, 생성 후 `OO님의 서재 들어가기` 버튼
- 최초 입장 시 이름(필수), 나이·성별·생일(선택)을 입력하는 로컬 프로필 모달
- `프로필과 기록은 이 브라우저에만 저장됩니다` 안내
- `백업에서 복원하기` 보조 링크

MVP에서는 복잡한 인트로 애니메이션을 제외하고, 정적 일러스트에 빛 입자와 잎 흔들림 정도만 CSS로 표현한다.

### 7.2 메인 화면 — 나의 서재

구성:

- 분홍빛 세계수 원본 일러스트 배경
- 세계수가 은은하게 비치는 반투명·블러 달력 패널
- 현재 연·월과 이전/다음 달 이동
- 월간 달력
- `달력 보기 / 목록 보기` 전환
- 날씨 아이콘과 온도
- AI 서재 관리자 캐릭터
- 상단 로컬 프로필 이름과 프로필 수정 진입 버튼

날짜 상태:

- 일기 없음: 비어 있는 날짜
- 일기 있음: 기분 아이콘 + 저장 당시 날씨 아이콘 + 제목 표시
- 날씨 모름: 날씨 아이콘 자리를 비우고 기분 아이콘과 제목만 표시
- 오늘: 테두리 강조
- 선택 날짜: 포근한 발광 효과

### 7.3 AI 캐릭터 메뉴

캐릭터 클릭 시 다음 세 가지 진입점을 보여준다.

1. 오늘의 일기 쓰기
2. 지난 일기 찾아보기
3. 지난 날짜 일기 추가하기

오늘 일기는 왼쪽 패널에서 AI 또는 수동으로 작성할 수 있다. 과거 날짜 추가는 `모아 → 지난 날짜 일기 추가`를 기본 진입점으로 사용한다. 중복되는 플로팅 `+` 버튼과 달력 하단 `+[날짜] 기록 추가` 버튼은 제공하지 않는다.

### 7.4 AI 대화 화면

- 캐릭터 초상과 대화 말풍선
- 메시지 입력창
- 사진/파일 첨부 버튼과 전송 전 실제 이미지 미리보기
- `친근하게 / 예의바르게 / 정중하게` 말투 선택
- `오늘은 여기까지, 일기로 정리해줘` 버튼
- 대화 종료 확인
- 생성 중 상태와 오류 재시도

첫 질문 예시:

- “오늘 가장 오래 머릿속에 남은 장면이 있나요?”
- “별일 없었다고 느껴진다면, 집을 나선 뒤 처음 기억나는 순간부터 이야기해도 좋아요.”

AI는 Gemini의 일반적인 대화 능력을 유지해 잡담, 공감, 의견, 정보와 질문을 맥락에 맞게 자유롭게 주고받는다. 매번 질문으로 끝내거나 일기 작성을 성급히 유도하지 않는다. 사용자가 종료 의사를 보이면 일기 초안을 만든다. 선택한 말투는 이후 응답에 즉시 반영하고 브라우저 설정에 저장한다. 프로필 이름은 자연스러운 순간에만 사용하며 매 답변마다 반복하지 않는다.

사용자가 채팅으로 사진, PDF 또는 TXT를 보내면 첨부물이 AI에 전송된다는 사실을 전송 버튼 근처에 안내한다. AI는 사용자 설명과 지원 파일의 내용을 함께 참고하되, 사진만 보고 알 수 없는 사실은 추측하지 않는다. DOCX, HWP, ZIP 등 MVP에서 분석하지 않는 형식은 최종 일기에 첨부만 한다.

### 7.5 일기 초안/편집 화면

- 날짜
- 제목
- 본문 편집기
- 선택 기분
- 저장된 날씨
- 첨부 파일 추가/삭제
- `저장`, `다시 생성`, `취소`

AI가 생성한 글임을 작게 표시하되 사용자가 수정한 최종 본문을 정본으로 저장한다. `다시 생성`은 현재 대화를 바탕으로 새 초안을 만들며 기존 수정본을 덮기 전에 확인한다.

### 7.6 일기 상세 화면

- 제목, 날짜, 본문, 기분, 당시 날씨
- 첨부 이미지 미리보기 및 파일 열기
- 수정/삭제
- `이 날의 대화 보기`: 당시 사용자·AI 대화 텍스트를 읽기 전용으로 표시하며 추가 입력창은 열지 않음
- 이전/다음 기록 이동

### 7.7 지난 기록 검색 화면

사용자 질문 예시:

- “2026년 6월에 알바 때문에 힘들었던 날 찾아줘.”
- “벚꽃을 본 날이 언제였지?”
- “2025년 10월의 행복했던 기억을 보여줘.”
- “6월 기록을 보여줘.”
- “29일에 쓴 일기 찾아줘.”
- “비 오는 날이나 울적했던 날을 보여줘.”

검색 대상은 날짜, 제목, 본문, 저장된 대화 텍스트, 기분과 날씨다. `29`, `29일`, `6월`, `6월의 기록`, `6월 29일`, `2026년 6월`, `2026-06-29` 형식을 인식한다. 기분과 날씨는 `울적했던 날`, `피곤했던 날`, `비 오는 날`, `맑았던 날`, `날씨 모름` 같은 표현과 연결한다. 결과는 근거 일기 카드와 날짜를 표시하며, 결과가 없으면 기억을 지어내지 않는다. 사진 자체의 시각적 내용 검색은 아직 지원하지 않으며, AI가 대화에서 사진 내용을 언급한 텍스트가 있을 때만 간접 검색될 수 있다.

## 8. 주요 사용자 흐름

### 흐름 A — AI로 오늘 일기 작성

시작 화면 → 최초 이용 시 로컬 프로필 생성 → OO님의 서재 들어가기 → 캐릭터 클릭 → 말투 선택 → 오늘의 일기 쓰기 → 대화 및 필요 시 첨부 전송 → 일기 초안 생성 → 사용자 수정 → 저장 → 달력에 기분·날씨 아이콘과 제목 표시

### 흐름 B — 수동으로 오늘/과거 일기 작성

오늘: 메인 왼쪽 패널 → 직접 일기 쓰기 → 첨부 추가 → 저장

과거: 모아 클릭 → 지난 날짜 일기 추가 → 날짜 선택/편집 → 첨부 추가 → 날씨 모름 또는 직접 지정 → 저장

### 흐름 C — 지난 일기 탐색

메인 → 달력 날짜 클릭 또는 목록 보기 → 일기 상세 → 필요 시 수정

### 흐름 D — AI로 지난 기록 찾기

캐릭터 클릭 → 지난 일기 찾아보기 → 자연어 질문 → 날짜·키워드·대화·기분·날씨 검색 → 관련 기록 카드 → 선택한 기록 상세

### 흐름 E — 전체 기록 백업

설정 → 전체 백업 만들기 → 백업 구성 확인 → ZIP 파일 생성 → 사용자가 안전한 위치에 저장

### 흐름 F — 백업 복원

시작 화면 또는 설정 → 백업에서 복원하기 → ZIP 선택 → 형식과 버전 검증 → 복원될 항목 수 확인 → 현재 데이터 자동 백업 → 전체 교체 확인 → 복원 → 달력 재구성

### 흐름 G — 로컬 프로필 수정

상단 프로필 버튼 또는 설정 → 프로필 수정 → 이름 및 선택 기본 정보 변경 → 저장 → 이후 AI 대화부터 새 프로필 문맥 적용

## 9. 기능 요구사항과 완료 조건

### FR-01 로컬 프로필 입장

- 외부 계정 회원가입, 비밀번호 인증과 OAuth 로그인을 제공하지 않는다.
- 최초 진입 시 이름을 필수로 입력하고 나이·성별·생일은 선택할 수 있다.
- 프로필 생성 후 시작 화면의 입장 버튼에 사용자 이름을 표시한다.
- 프로필은 설정과 상단 프로필 버튼에서 수정할 수 있다.
- 최초 진입 시 프로필과 기록이 현재 기기와 브라우저에만 저장된다는 점을 안내한다.
- 공용 PC에서 사용할 경우 같은 브라우저 사용자에게 기록이 노출될 수 있음을 안내한다.

완료 조건: 로컬 프로필을 만든 뒤 첫 일기를 작성할 수 있고, 앱을 닫았다가 같은 브라우저로 다시 열어도 프로필과 기록이 남아 있다.

### FR-02 일기 CRUD

- 현재 브라우저 저장소에서 날짜별 일기 1개를 기본으로 한다.
- 같은 날짜에 다시 작성하면 새 글 생성이 아니라 기존 일기를 열어 수정하도록 안내한다.
- 제목이 비어 있으면 날짜 기반 기본 제목을 사용할 수 있다.
- 삭제 전 확인 모달을 표시한다.

완료 조건: 작성, 새로고침 후 조회, 수정, 삭제가 모두 동작하며 달력 상태가 즉시 반영된다.

### FR-03 AI 대화 및 초안 생성

- 대화는 세션 단위로 저장한다.
- 사용자가 버튼 또는 자연어로 종료를 요청하면 일기 초안을 만든다.
- 생성 결과는 `제목`과 `본문` 구조로 반환한다.
- AI는 대화에 없는 사건을 추가하지 않도록 프롬프트에서 제한한다.
- 실패하면 대화와 입력을 보존한 채 재시도할 수 있다.
- 사용자는 대화 전이나 도중에 AI 말투를 변경할 수 있다.
- 선택 말투는 설정에 저장하며 새 대화의 기본값으로 사용한다.
- AI 요청에는 로컬 프로필을 함께 전달하고 이름을 자연스러운 호칭으로 사용한다.
- 나이·성별·생일은 관련 맥락에서만 참고하며 감정이나 성격을 추측하는 근거로 사용하지 않는다.
- AI의 응답을 일기 질문으로만 제한하지 않고 Gemini의 일반 대화 능력을 유지한다.

완료 조건: 대화 길이에 관계없이 확인된 내용을 바탕으로 한국어 제목과 본문 초안을 생성하고, 세 말투의 응답 차이를 확인할 수 있으며, 사용자가 수정 후 저장할 수 있다.

### FR-04 첨부 파일

- 채팅 메시지와 수동 편집 화면에서 이미지와 일반 파일을 추가할 수 있다.
- MVP 권장 허용 형식: JPG, PNG, WEBP, PDF, TXT.
- 파일당 최대 10MB, 일기당 최대 5개로 제한한다.
- 이미지는 미리보기하고 그 외 파일은 이름과 크기를 표시한다.
- 채팅 전송 전, 채팅 기록, 편집 화면과 최종 일기에서 이미지는 실제 이미지로 표시한다.
- 채팅에 보낸 JPG, PNG, WEBP, PDF, TXT는 사용자 설명과 함께 Gemini 입력에 포함할 수 있다.
- AI가 파일 내용을 참고했다는 상태를 화면에 표시한다.
- 대화에서 사용한 첨부물은 최종 일기에 같은 첨부물 ID로 연결하고 중복 저장하지 않는다.
- 일기 또는 대화 삭제 시 다른 연결이 없는 첨부물만 함께 삭제한다.

완료 조건: 채팅에 올린 지원 파일의 내용이 최종 초안에 반영되고, 새로고침 후에도 최종 일기에서 같은 첨부물을 열 수 있다.

### FR-05 달력/목록

- 달력은 월 단위로 탐색할 수 있다.
- 일기가 있는 날짜를 시각적으로 구분한다.
- 일기가 있는 날짜에는 잎 아이콘 대신 기분·날씨 아이콘과 제목을 표시한다.
- 저장 날씨가 모름이면 날씨 아이콘은 표시하지 않는다.
- 목록은 최신 날짜순이며 제목과 본문 일부를 표시한다.
- 달력과 목록은 같은 데이터 원본을 사용한다.
- 중복 작성 동선인 플로팅 `+` 버튼과 달력 하단 기록 추가 버튼은 제거한다.

완료 조건: 보기 전환 뒤에도 선택 월과 검색 상태가 유지된다.

### FR-06 날씨 배경

- 위치 권한에 동의하면 현재 좌표로 날씨를 조회한다.
- 거부하면 서울 기본값 또는 마지막으로 선택한 지역을 사용한다.
- 날씨 코드를 맑음, 흐림, 비, 눈의 네 상태로 단순화한다.
- 날씨 API 실패 시 기본 맑음 배경을 사용하고 앱 이용을 막지 않는다.
- 오늘 일기는 현재 날씨를 기본값으로 저장하되 사용자가 수정할 수 있다.
- 과거 날짜 일기는 `날씨 모름`을 기본값으로 사용하고 사용자가 맑음·흐림·비·눈 중 직접 지정할 수 있다.

완료 조건: 날씨 응답에 따라 배경 레이어와 아이콘이 함께 바뀐다.

### FR-07 서재 관리자의 기록 검색

MVP 검색은 AI가 모든 기록을 통째로 읽는 구조가 아니라 브라우저 내부 검색으로 동작한다.

1. IndexedDB에서 일기와 저장된 대화 텍스트를 읽는다.
2. 날짜, 제목, 본문, 대화, 기분과 날씨 문자열을 하나의 검색 대상으로 구성한다.
3. 한국어 날짜 표현은 연·월·일 조건으로 해석해 해당 날짜와 모두 일치하는 기록만 남긴다.
4. 기분과 날씨는 미리 정의한 자연어 별칭으로 연결한다.
5. 근거가 된 일기 카드와 날짜를 반환한다.

지원 날짜 예시는 `29`, `29일`, `6월`, `6월의 기록`, `6월 29일`, `2026년 6월`, `2026년 6월 29일`, `2026-06-29`다. 기분·날씨 예시는 `울적`, `피곤`, `설렘`, `비 오는 날`, `맑았던 날`, `눈 온 날`, `날씨 모름`이다. 의미가 비슷하지만 정의되지 않은 표현까지 찾는 벡터 검색과 사진 시각 내용 검색은 후속 버전으로 미룬다.

완료 조건: 한국어 날짜, 키워드, 저장 대화, 기분 또는 날씨가 포함된 질문에서 관련 일기 카드를 반환하며, 결과가 없을 때 존재하지 않는 기록을 만들지 않는다.

### FR-08 전체 백업과 복원

- 백업은 일기, 대화, 설정, 로컬 프로필, 날씨 스냅샷, 사진과 파일을 모두 포함한다.
- 백업 파일은 `manifest.json`, 구조화 데이터, 첨부 원본을 포함한 단일 ZIP으로 생성한다.
- 파일명 예시: `worldtree-diary-backup-2026-07-20.zip`.
- `manifest.json`에는 백업 형식 버전, 앱 버전, 생성 시각과 항목 수를 기록한다.
- 복원 전 파일 구조, 버전, 필수 필드와 첨부 파일 누락 여부를 검사한다.
- MVP 복원 방식은 기존 데이터와 병합하지 않고 전체 교체로 한다.
- 전체 교체 전에 현재 데이터의 자동 백업 파일을 먼저 생성하도록 안내한다.
- 잘못된 파일이나 지원하지 않는 새 버전은 데이터를 변경하지 않고 거부한다.
- 백업 ZIP은 암호화되지 않으므로 민감한 파일이라는 경고를 표시한다.

완료 조건: 첨부물이 있는 일기를 백업하고 브라우저 데이터를 초기화한 뒤 복원했을 때 일기, 대화, 설정과 첨부물이 동일하게 돌아온다.

### FR-09 지난 대화 열람

- AI로 작성한 일기에는 완성된 대화 세션을 연결한다.
- 일기 상세의 `이 날의 대화`는 연결된 세션의 사용자·AI 메시지 텍스트를 시간순으로 표시한다.
- 지난 대화 화면은 읽기 전용이며 메시지 입력창이나 전송 버튼을 제공하지 않는다.
- 연결된 대화가 없거나 텍스트가 남아 있지 않으면 새 채팅을 만들지 않고 안내 메시지를 표시한다.

완료 조건: 저장된 일기에서 당시 대화 원문을 다시 볼 수 있고, 열람 과정에서 새 세션이나 메시지가 생성되지 않는다.

## 10. AI 동작 규칙

### 서재 관리자 성격

- 따뜻하고 조용하지만 지나치게 감상적이지 않다.
- 평범한 일을 가치 없다고 판단하지 않는다.
- 사용자의 표현을 존중하고 문체를 과하게 바꾸지 않는다.
- 해결책을 성급히 제시하기보다 기록을 돕는다.
- AI이며 의료·심리 전문가가 아님을 숨기지 않는다.
- 로컬 프로필 이름을 자연스러운 순간에 사용하되 매 응답마다 반복하지 않는다.
- 선택 입력된 나이·성별·생일을 관련 맥락에서만 조심스럽게 참고한다.

### 선택 가능한 대화 말투

말투 선택은 서재 관리자의 대화 표현에만 적용한다. 최종 일기는 어느 말투를 선택했든 사용자의 1인칭 말투와 감정 강도를 유지한다.

| 선택 | 동작 원칙 | 질문 예시 |
|---|---|---|
| 친근하게 | 가까운 친구처럼 편안한 반말, 짧고 자연스러운 반응 | “오늘은 어땠어? 기억에 남는 장면부터 이야기해줄래?” |
| 예의바르게 | 따뜻하고 부담 없는 해요체, 기본 선택값 | “오늘은 어땠어요? 기억에 남는 장면부터 이야기해 주실래요?” |
| 정중하게 | 차분하고 절제된 존댓말, 가벼운 표현과 이모티콘 최소화 | “오늘 하루 중 가장 기억에 남는 장면을 말씀해 주시겠어요?” |

세 말투 모두 질문 수, 공감 수준, 안전 규칙과 사실성 기준은 동일하다. 말투를 바꾸더라도 AI의 성격이나 기억이 바뀐 것처럼 표현하지 않는다.

### 일기 생성 프롬프트 원칙

- 입력 대화에 나온 사실만 사용한다.
- 사용자가 전송한 지원 첨부물에서 확인되는 내용과 사용자의 설명만 사용한다.
- 1인칭 시점으로 쓴다.
- 사용자의 말투와 감정 강도를 유지한다.
- 억지 교훈, 과장된 위로, 진단 표현을 피한다.
- 출력은 파싱 가능한 JSON 형태로 제한한다.

권장 출력 형태:

```json
{
  "title": "오늘의 작은 친절",
  "content": "...",
  "mood": "calm"
}
```

### 안전 처리

- 서비스 소개와 대화 화면에 “전문 상담이나 응급 서비스를 대체하지 않는다”는 안내를 둔다.
- 자해·자살 등 즉각적 위기 표현이 감지되면 일기 질문을 계속하지 않고 긴급 도움 안내를 우선한다.
- 한국 사용자에게는 24시간 자살예방 상담전화 `109`와 즉각적인 위험 시 `112` 또는 `119`를 안내한다.
- AI가 사용자를 진단하거나 약물·치료 지시를 하지 않게 한다.

## 11. 데이터 저장 설계

### 11.1 권장안

**외부 계정 인증 없이 현재 브라우저의 IndexedDB에 로컬 프로필, 모든 기록과 첨부물을 저장한다.**

- 일기, 대화, 설정과 첨부 메타데이터: IndexedDB
- 사진과 파일 원본: IndexedDB의 Blob
- 화면의 일시 상태: React state
- 작고 민감하지 않은 UI 초기값만 localStorage 사용 가능
- Gemini 호출: API 키를 환경 변수로 가진 로컬 Node.js 서버 프록시
- 날씨 조회: 브라우저에서 Open-Meteo 직접 호출

localStorage는 문자열 중심의 작은 저장소이므로 일기 정본과 파일 저장에 사용하지 않는다. IndexedDB는 구조화 데이터와 Blob을 함께 보관하며, 앱 코드에서는 스키마 버전 관리와 트랜잭션을 단순화하기 위해 Dexie 사용을 권장한다.

이 구조는 서버 DB 계정이 필요 없고 기존 기록 조회와 수동 작성은 오프라인에서도 가능하다. 단, Gemini 대화와 파일 분석, 현재 날씨 갱신에는 인터넷 연결이 필요하다.

IndexedDB 데이터는 브라우저 창을 닫거나 컴퓨터를 재부팅해도 유지된다. 다만 같은 브라우저 프로필과 같은 origin으로 접속해야 하며, 사이트 데이터 삭제, 시크릿 모드 종료, 브라우저 프로필 초기화, 기기 고장 또는 브라우저의 저장 공간 정리가 발생하면 손실될 수 있다. 앱의 실행 주소와 포트는 고정하고 장기 보관은 ZIP 백업을 기준으로 한다.

### 11.2 데이터 모델

#### `settings`

| 필드 | 형식 | 설명 |
|---|---|---|
| key | string PK | 설정 키 |
| value | any | 설정 값 |
| updated_at | ISO datetime | 수정 시각 |

주요 설정: `profile`, `chatTone`, `viewMode`, `weather`, `lastBackupAt`.

`profile` 값 구조:

| 필드 | 형식 | 설명 |
|---|---|---|
| name | string | AI가 부를 이름, 필수, 최대 20자 |
| age | number nullable | 1~120세, 선택 |
| gender | string nullable | `female`, `male`, `nonbinary`, `private`, 선택 |
| birthday | date string nullable | 생일, 선택 |
| updated_at | ISO datetime | 프로필 수정 시각 |

#### `diaries`

| 필드 | 형식 | 설명 |
|---|---|---|
| id | uuid PK | 일기 ID |
| diary_date | date string | 현지 일기 날짜 |
| title | string | 제목 |
| content | string | 최종 본문 |
| source | string | `ai` 또는 `manual` |
| mood | string nullable | 선택 기분 |
| weather | object nullable | 날씨 코드, 온도, 지역 스냅샷 |
| attachment_ids | uuid[] | 연결 첨부물 |
| created_at | ISO datetime | 생성 시각 |
| updated_at | ISO datetime | 수정 시각 |

제약: `diary_date`는 한 날짜에 한 일기만 존재하도록 unique index를 둔다.

#### `chat_sessions`

| 필드 | 형식 | 설명 |
|---|---|---|
| id | uuid PK | 대화 세션 ID |
| diary_date | date string | 작성 대상 날짜 |
| diary_id | uuid nullable | 완성된 일기 연결 |
| tone | string | 해당 세션의 말투 스냅샷 |
| status | string | `active`, `completed`, `abandoned` |
| created_at | ISO datetime | 시작 시각 |
| updated_at | ISO datetime | 수정 시각 |

#### `chat_messages`

| 필드 | 형식 | 설명 |
|---|---|---|
| id | uuid PK | 메시지 ID |
| session_id | uuid | 소속 세션 |
| role | string | `user`, `assistant`, `system` |
| content | string | 메시지 본문 |
| attachment_ids | uuid[] | 메시지에 포함된 첨부물 |
| created_at | ISO datetime | 생성 시각 |

#### `attachments`

| 필드 | 형식 | 설명 |
|---|---|---|
| id | uuid PK | 첨부 ID |
| blob | Blob | 파일 원본 또는 이미지 압축본 |
| original_name | string | 원본 파일명 |
| mime_type | string | MIME 형식 |
| byte_size | number | 파일 크기 |
| ai_readable | boolean | MVP AI 분석 지원 형식 여부 |
| analysis_status | string | `none`, `pending`, `used`, `failed` |
| created_at | ISO datetime | 추가 시각 |

하나의 첨부물 ID를 채팅 메시지와 최종 일기에서 함께 참조한다. 어느 곳에서도 참조되지 않는 첨부물은 고아 데이터 정리 대상이다.

#### `backup_manifest`

IndexedDB 테이블이 아니라 ZIP 안에 포함되는 메타데이터 파일이다.

| 필드 | 형식 | 설명 |
|---|---|---|
| format | string | `worldtree-diary-backup` |
| format_version | number | 백업 스키마 버전 |
| app_version | string | 생성 앱 버전 |
| exported_at | ISO datetime | 백업 시각 |
| counts | object | 일기, 대화, 메시지, 첨부물 개수 |
| timezone | string | 기본 `Asia/Seoul` |

### 11.3 보안 규칙

- 로그인이나 서버 권한 분리가 없으므로 같은 브라우저 프로필을 사용하는 사람은 기록을 볼 수 있다.
- 공용 PC와 시크릿 모드 사용을 권장하지 않는다는 안내를 제공한다.
- Gemini API 키는 프론트엔드 코드나 Git 저장소에 넣지 않는다.
- 로컬 Node.js 프록시는 키를 `.env`에서 읽고 Gemini 요청만 전달한다.
- 로컬 서버 로그에 대화, 일기, 파일 내용 또는 API 키를 남기지 않는다.
- 과거 기록 검색은 현재 브라우저 안에서 수행하며 검색을 위해 전체 일기장을 Gemini에 전송하지 않는다.
- `모든 로컬 데이터 삭제` 기능은 이중 확인 후 하나의 IndexedDB 트랜잭션으로 수행한다.
- 백업 파일은 암호화되지 않은 민감정보이므로 공유하지 않도록 경고한다.

### 11.4 백업과 용량

브라우저 저장 용량과 삭제 정책은 브라우저와 기기 환경에 따라 달라진다. 앱 시작 시 영구 저장 권한을 요청할 수 있도록 `navigator.storage.persist()`를 사용하고, `navigator.storage.estimate()`로 사용량을 설정 화면에 표시한다. 이는 삭제 가능성을 줄일 뿐 기기 고장이나 사용자의 사이트 데이터 삭제까지 막지는 못한다.

- 첫 일기 저장 후 백업 기능을 안내한다.
- 마지막 백업 시각을 설정 화면과 메인 화면 경고에 표시한다.
- 30일 이상 백업하지 않았거나 저장 공간 사용률이 높으면 경고한다.
- 사진은 저장 전에 긴 변 1600px 정도로 축소하고 품질을 조정한다.
- 백업 ZIP은 구조화 데이터와 `attachments/` 원본 파일을 포함한다.
- 복원은 임시 메모리에서 전체 검증한 뒤 하나의 IndexedDB 트랜잭션으로 적용한다.
- 백업 생성 실패나 복원 검증 실패 시 기존 데이터를 변경하지 않는다.

## 12. 권장 기술 구성

- 프론트엔드: React 19 + TypeScript + Vinext(Vite 기반)
- UI: 전역 CSS + Lucide React 아이콘
- 로컬 데이터와 파일: IndexedDB + Dexie
- 백업 ZIP 생성/검증: fflate 또는 JSZip
- AI 프록시: `/api/chat` 서버 API 라우트, Gemini 키는 `.env`에 보관
- AI: 사용 가능한 최신 Gemini Flash 무료 모델
- 날씨: Open-Meteo Current Weather API
- 실행: `127.0.0.1:5173` 로컬 개발 서버에서 프론트엔드와 AI API 라우트를 함께 실행

프론트엔드는 반응형 웹 앱이지만 기록의 정본은 현재 브라우저에만 존재한다. 다른 기기나 다른 브라우저 프로필에서는 백업 파일을 직접 복원해야 한다.

Gemini 모델 ID와 API 키는 코드 여러 곳에 직접 쓰지 않고 서버 환경 변수로 관리한다. 모델 지원 여부와 무료 한도는 바뀔 수 있으므로 실행 전 AI Studio에서 실제 할당량을 확인한다. 나중에 인터넷에 공개 배포하려면 로컬 프록시를 인증·속도 제한이 적용된 호스팅 서버 함수로 교체해야 한다.

## 17. 확정 및 추가 결정 사항

### 결정 1 — 하루 한 편인가, 여러 편인가

MVP 권장: 하루 한 편. 대화와 수동 기록을 한 일기에 계속 보완하도록 한다. 여러 편을 허용하면 달력 표시, 검색 결과, 대화 연결 구조가 복잡해진다.

### 결정 2 — AI 대화 원문을 저장하는가

서재 관리자가 과거 대화도 찾고 사용자가 당시 맥락을 다시 볼 수 있어야 하므로 저장한다. 완성된 일기에서 대화 원문은 읽기 전용 화면으로 열며 새 채팅으로 이어지지 않는다. 대화 원문은 최종 일기보다 더 민감할 수 있으므로 `대화 원문도 저장됩니다`라는 안내와 후속 세션 삭제 기능이 필요하다.

### 결정 3 — AI가 어떤 말투로 대화하는가

MVP 확정: 친근하게, 예의바르게, 정중하게 세 가지 중에서 선택한다. 기본값은 `예의바르게`이며 설정과 대화 세션에 저장한다. 이 선택은 채팅 말투에만 적용하고 일기 초안은 사용자의 말투를 유지한다.

### 결정 4 — 위치 정보 사용 방식

MVP 권장: 매 방문마다 브라우저 위치 권한을 요청하지 않는다. 최초 동의 후 좌표를 기기 설정으로 저장하거나, 거부 시 서울을 기본값으로 사용한다. 일기에는 정확한 좌표가 아니라 도시명과 날씨만 저장한다.

### 결정 5 — 서비스 수준

MVP 확정: 외부 계정 인증 없이 하나의 로컬 프로필을 사용하는 로컬 저장 웹 앱으로 개발한다. 로컬 프로필은 보안 로그인이나 사용자별 권한 분리가 아니다. 현재 브라우저가 유일한 정본이며 자동 동기화는 제공하지 않는다. 인터넷 공개 서비스로 전환할 때에는 실제 인증, AI 프록시 보호, 개인정보 처리방침, 파일 검사와 클라우드 동기화를 다시 설계한다.

### 결정 6 — 백업 복원 방식

MVP 확정: 병합이 아닌 전체 교체만 지원한다. 복원 전에 현재 상태의 백업 생성을 안내하고, 선택한 백업을 완전히 검증한 뒤에만 기존 데이터를 교체한다.

### 결정 7 — 중복 일기 작성 버튼

MVP 확정: 플로팅 `+` 버튼과 달력 하단 `+[날짜] 기록 추가` 버튼을 제거한다. 오늘 수동 작성은 왼쪽 패널, 과거 날짜 작성은 모아의 `지난 날짜 일기 추가` 메뉴를 사용한다.

### 결정 8 — 달력의 기록 표시

MVP 확정: 잎 아이콘 대신 기분 아이콘과 저장된 날씨 아이콘을 제목 앞에 표시한다. 날씨 모름은 별도 물음표를 표시하지 않고 날씨 아이콘을 생략한다.

### 결정 9 — 사진 내용 검색

현재 MVP에서는 사진 파일 자체를 매번 다시 분석하지 않는다. 사진 내용이 대화나 일기 본문에 텍스트로 남아 있으면 간접 검색할 수 있다. 사진 업로드 시 Gemini 설명·태그를 별도 저장해 검색하는 기능은 P1/P2 후속 범위로 둔다.

## 19. 개인정보와 Gemini 무료 플랜 주의

일기는 매우 민감한 개인정보가 될 수 있다. 로컬 저장은 일기 정본이 별도 DB 서버에 올라가는 것을 막지만, Gemini 대화와 AI 분석을 허용한 첨부물, AI 문맥에 포함되는 로컬 프로필 정보는 Google 서버로 전송된다. 현재 Gemini API 가격 안내에는 무료 티어로 보낸 콘텐츠가 Google 제품 개선에 사용될 수 있다고 표시되어 있고, 유료 티어는 그렇지 않다고 구분되어 있다. 따라서 다음을 제품 요구사항으로 취급해야 한다.

- 첫 AI 사용 및 파일 전송 전에 AI 처리 사실과 외부 전송 범위를 알린다.
- 이름, 나이, 성별, 생일 중 입력한 프로필 정보가 AI 대화 요청에 포함된다는 점을 안내하고 선택 정보는 비워둘 수 있게 한다.
- AI 사용에 동의하지 않아도 수동 일기는 쓸 수 있게 한다.
- 과거 기록 검색은 브라우저 내부에서 수행하고 검색만을 위해 일기 내용을 Gemini에 전송하지 않는다.
- 지원하지 않는 첨부물은 Gemini에 보내지 않고 로컬 첨부로만 보관한다.
- 주민등록번호, 계좌, 상세 주소 등 민감정보를 입력하지 않도록 안내한다.
- 포트폴리오 데모에서는 실제 민감 일기 대신 테스트 데이터를 사용한다.
- 실제 공개 전에는 데이터 처리 조건을 다시 검토하고 필요하면 유료 티어 또는 별도 보호 방식을 선택한다.
