# 나무의 서재 PRD

- 문서 버전: v1.0
- 기준일: 2026-07-21
- 문서 상태: 구현 완료 기준
- 제품 형태: 로컬 프로필 기반 반응형 웹 앱
- 배포 대상: Vercel

## 1. 제품 정의

나무의 서재는 반복되는 일상에서 지나치기 쉬운 사건과 감정을 AI 서재지기 ‘모아’와 대화하거나 직접 작성해 날짜별 일기로 남기는 웹 애플리케이션이다.

사용자는 빈 편집기에서 처음부터 글을 완성하지 않아도 된다. 모아와 자연스럽게 대화한 뒤 AI가 만든 초안을 수정해 저장할 수 있으며, 직접 일기를 쓰는 방식도 선택할 수 있다. 저장된 기록은 달력과 목록으로 다시 열어보고 날짜, 본문, 대화, 기분, 날씨를 기준으로 검색할 수 있다.

## 2. 제품 배경

학교, 집, 아르바이트처럼 비슷한 하루가 반복되면 시간이 지난 뒤 남아 있는 기억이 거의 없다고 느낄 수 있다. 나무의 서재는 특별한 사건만 기록할 가치가 있다는 부담을 줄이고, 평범한 하루의 작은 장면도 사라지지 않게 보관하는 것을 목적으로 한다.

제품이 제공하는 핵심 가치는 다음과 같다.

1. 대화로 기록의 시작 부담을 낮춘다.
2. AI 초안을 사용자가 직접 다듬어 최종 기록의 주도권을 유지한다.
3. 사진과 파일, 당시 대화까지 하루의 맥락으로 함께 보관한다.
4. 달력과 검색을 통해 지나간 기억을 다시 찾기 쉽게 만든다.

## 3. 현재 제품 범위

### 3.1 제공 기능

- 외부 계정 인증이 없는 단일 로컬 프로필
- 이름, 나이, 성별, 생일 설정과 수정
- Gemini 기반 자유 대화
- 친근하게, 예의바르게, 정중하게의 세 가지 대화 말투
- 대화 내용을 바탕으로 한 일기 초안 생성
- 수동 일기 작성
- 날짜별 일기 조회, 수정, 삭제
- 과거 날짜 일기 작성
- 기분과 날씨 기록
- 채팅 및 일기 편집 화면의 사진·파일 첨부
- 이미지 미리보기와 일반 파일 다운로드
- 완료된 일기에 연결된 과거 대화 텍스트 열람
- 월간 달력과 전체 목록 보기
- 날짜, 제목, 본문, 대화, 기분, 날씨의 로컬 검색
- 현재 날씨 조회와 시작 화면 날씨 효과
- IndexedDB 로컬 저장
- 사진과 파일을 포함한 ZIP 전체 백업·복원
- 브라우저 저장 공간 사용량과 영구 저장 요청 상태 표시
- 모든 로컬 데이터 삭제
- 데스크톱과 모바일 화면 대응

### 3.2 현재 운영 범위

- 로컬 프로필은 실제 로그인이나 사용자 인증이 아니다.
- 기록은 서버 계정이 아니라 현재 브라우저의 IndexedDB에 저장된다.
- 다른 브라우저나 기기로 기록이 자동 동기화되지 않는다.
- 지난 기록 검색은 Gemini가 아닌 브라우저 내부 검색으로 실행된다.
- 사진 자체의 시각 정보를 별도 태그로 저장하거나 검색하지 않는다.
- 백업 ZIP은 암호화되지 않는다.

## 4. 주요 사용자

- 반복되는 학교·직장·아르바이트 생활을 기록하고 싶은 사람
- 글을 처음부터 쓰는 것보다 대화로 하루를 떠올리는 것이 편한 사람
- 사진과 대화를 포함해 당시의 기억을 다시 보고 싶은 사람
- 계정 가입 없이 한 기기에서 개인 일기장을 사용하고 싶은 사람

## 5. 제품 원칙

1. 평범한 하루를 과장하거나 억지로 교훈화하지 않는다.
2. AI가 만든 결과는 초안이며 최종 제목과 본문은 사용자가 수정한다.
3. 모아는 질문만 반복하지 않고 잡담, 공감, 의견과 질문을 문맥에 맞게 주고받는다.
4. 프로필 이름은 자연스러운 순간에만 사용하고 매 답변마다 반복하지 않는다.
5. 일기, 대화와 첨부물의 저장 위치와 외부 AI 전송 범위를 화면에서 알린다.
6. AI와 날씨 연결이 실패해도 저장된 기록 조회와 수동 작성 흐름을 유지한다.

## 6. 정보 구조

### 6.1 시작 화면

시작 화면은 분홍빛 세계수 배경과 다음 요소로 구성된다.

- 서비스명 ‘나무의 서재’
- 제품 소개 문구
- 프로필 생성 또는 사용자 이름이 포함된 서재 입장 버튼
- ZIP 백업 복원 버튼
- 로컬 저장 안내
- 현재 날씨, 온도와 지역 표시
- 시작 화면 배경 효과 선택기

배경 효과는 현재 날씨 자동 반영, 화창, 흐림, 비, 눈 중 선택할 수 있다. 선택값은 브라우저 설정에 저장된다.

- 화창: 화면 곳곳의 반복 반짝임
- 흐림: 화면이 약간 어두워지고 하단의 회색 안개 흐름 표시
- 비: 빗줄기 낙하
- 눈: 눈송이 낙하

날씨 애니메이션은 시작 화면에만 표시된다.

### 6.2 로컬 프로필 화면

최초 입장 시 프로필 생성 모달을 연다.

- 이름: 필수, 최대 20자
- 나이: 선택, 1~120
- 성별: 선택하지 않음, 여성, 남성, 기타/논바이너리, 공개하지 않음
- 생일: 선택, 미래 날짜 입력 제한

저장한 프로필은 시작 화면 입장 버튼, 메인 상단과 설정 화면에 표시된다. 설정한 정보는 AI 요청에도 포함되며, 이름은 대화 호칭에 사용된다.

### 6.3 메인 화면

메인 화면은 세계수 배경 위에 반투명 유리 질감 패널을 배치한다.

상단 영역:

- 시작 화면으로 돌아가는 브랜드 버튼
- 현재 날씨, 온도와 위치 버튼
- 프로필 수정 버튼
- 로컬 보관 상태와 설정 버튼

오늘 패널:

- 현재 날짜와 요일
- 오늘 일기 작성 여부
- 모아와 오늘 이야기하기
- 직접 일기 쓰기
- 전체 일기 수와 기록이 존재하는 월 수

기록 패널:

- 이전·다음 달 이동
- 달력·목록 전환
- 날짜별 기록 표시
- 기록 상세 진입

### 6.4 서재지기 모아 메뉴

화면 하단의 모아 캐릭터를 누르면 다음 기능을 연다.

1. 오늘의 일기 쓰기
2. 지난 기억 찾아보기
3. 선택한 날짜에 지난 일기 추가

과거 날짜 일기를 작성할 때는 달력에서 날짜를 선택한 뒤 모아 메뉴의 ‘지난 날짜 일기 추가’를 사용한다. 달력 날짜를 두 번 눌러 편집 화면을 바로 열 수도 있다.

### 6.5 AI 대화 화면

대화 화면은 다음 요소로 구성된다.

- 작성 대상 날짜
- 세 가지 말투 선택
- 사용자와 모아의 말풍선
- 저장된 채팅 이미지와 파일 표시
- 파일 선택과 전송 전 미리보기
- 텍스트 입력과 전송 버튼
- AI 응답 대기 표시
- ‘오늘은 여기까지, 일기로 정리해줘’ 버튼
- 로컬 저장 및 Google 처리 안내

사용자가 대화창을 닫아도 활성 세션과 저장된 메시지는 IndexedDB에 남는다. 같은 날짜의 활성 세션을 다시 열면 기존 메시지를 이어서 표시한다.

### 6.6 일기 편집 화면

편집 화면에서 다음 값을 입력하거나 수정한다.

- 제목
- 본문
- 기분
- 사진과 파일
- 날씨

본문은 필수다. 제목이 비어 있으면 해당 날짜를 사용한 기본 제목을 자동 생성한다.

기분은 따뜻, 평온, 지침, 울적, 설렘 중 선택한다. 기본값은 평온이다.

오늘 일기는 현재 날씨가 기본으로 설정된다. 과거 날짜는 날씨 모름이 기본이며 사용자가 맑음, 흐림, 비, 눈 중 직접 선택할 수 있다. 직접 선택한 과거 날씨에는 온도를 저장하지 않는다.

### 6.7 일기 상세 화면

상세 화면은 다음 정보를 표시한다.

- 날짜와 제목
- 본문
- 기분
- 저장 당시 날씨 또는 날씨 모름
- AI 작성 또는 직접 작성 구분
- 첨부 이미지와 파일
- 이 날의 대화
- 수정과 삭제 버튼

‘이 날의 대화’는 해당 일기를 만들 때 저장된 사용자와 AI의 텍스트만 읽기 전용으로 표시한다. 입력창이나 추가 대화 기능은 제공하지 않는다.

### 6.8 설정 화면

설정 화면은 다음 기능을 제공한다.

- 로컬 프로필 수정
- 저장 공간 사용량 표시
- 브라우저 영구 저장 요청 허용 여부 표시
- 마지막 백업 날짜 표시
- ZIP 백업 생성
- ZIP 백업 복원
- 모든 로컬 데이터 삭제

모든 데이터 삭제는 두 번 확인한 뒤 IndexedDB를 삭제하고 앱을 다시 불러온다.

## 7. 핵심 사용자 흐름

### 흐름 A — 최초 입장

시작 화면 → 프로필 만들고 들어가기 → 이름과 선택 정보 입력 → 내 서재로 들어가기 → 메인 화면

### 흐름 B — AI로 오늘 일기 작성

메인 화면 → 모아와 오늘 이야기하기 → 말투 선택 → 대화와 필요 시 첨부 전송 → 일기 초안 생성 → 제목·본문·기분·날씨·첨부 확인 및 수정 → 서재에 보관

### 흐름 C — 직접 일기 작성

오늘은 오늘 패널의 ‘직접 일기 쓰기’로 진입한다. 과거 기록은 달력에서 날짜 선택 → 모아 메뉴 → 지난 날짜 일기 추가 순서로 진입한다.

### 흐름 D — 기록 다시 보기

달력의 기록이 있는 날짜 또는 목록의 일기 카드 선택 → 상세 화면 → 필요 시 대화 열람, 수정 또는 삭제

### 흐름 E — 지난 기록 검색

모아 메뉴 → 지난 기억 찾아보기 → 날짜 또는 검색어 입력 → 결과 카드 선택 → 일기 상세

### 흐름 F — 백업

설정 → ZIP 백업 만들기 → 브라우저가 전체 백업 파일 다운로드 → 마지막 백업 날짜 갱신

### 흐름 G — 복원

시작 화면 또는 설정 → ZIP 파일 선택 → 형식 확인 → 전체 교체 확인 → 기존 기록이 있으면 현재 상태 자동 백업 → ZIP 데이터 복원

## 8. 기능 요구사항

### FR-01 로컬 프로필

- 외부 회원가입, 비밀번호와 OAuth를 사용하지 않는다.
- 이름을 필수로 저장하고 나이, 성별, 생일은 선택 저장한다.
- 프로필을 생성한 이후 같은 브라우저에서 다시 열면 저장된 이름을 불러온다.
- 프로필은 상단 버튼과 설정 화면에서 수정할 수 있다.
- AI 요청에는 현재 프로필을 포함한다.

### FR-02 날짜별 일기

- 한 날짜에는 한 개의 일기만 저장한다.
- 같은 날짜를 다시 편집하면 기존 일기를 불러와 갱신한다.
- 본문이 없는 일기는 저장하지 않는다.
- 제목이 없으면 날짜 기반 제목을 사용한다.
- 일기에는 작성 방식, 기분, 날씨, 첨부물과 생성·수정 시각을 저장한다.
- 삭제 전 확인창을 표시하고 삭제 후 참조되지 않는 첨부물을 정리한다.

### FR-03 AI 대화

- 대화 세션과 메시지를 IndexedDB에 저장한다.
- 대화 말투는 친근하게, 예의바르게, 정중하게 중 선택한다.
- 선택한 말투는 전역 설정과 현재 세션에 저장한다.
- Gemini에는 최근 대화 기록, 현재 메시지, 지원 첨부물과 프로필을 전달한다.
- 일반 대화 응답은 정해진 문답 형식으로 제한하지 않는다.
- API 오류가 발생해도 먼저 저장한 사용자 메시지는 유지한다.
- API 키가 없으면 기본 데모 응답과 대화 기반 단순 초안을 반환한다.

### FR-04 AI 일기 초안

- 사용자가 한 번 이상 메시지를 남긴 뒤 초안 생성을 요청할 수 있다.
- 저장된 대화와 최대 5개의 AI 지원 첨부물을 다시 전송한다.
- Gemini 응답은 제목, 본문과 기분을 포함한 JSON으로 받는다.
- 대화와 첨부물에서 확인된 사실만 사용하도록 시스템 지침을 적용한다.
- 생성한 초안은 바로 저장하지 않고 편집 화면에서 사용자가 수정하게 한다.
- 최종 저장 시 연결 대화 세션을 완료 상태로 바꾸고 일기 ID를 연결한다.

### FR-05 첨부물

- 채팅은 JPG, PNG, WEBP, PDF, TXT 선택을 지원한다.
- 일기 편집 화면에서는 이미지와 일반 파일을 보관할 수 있다.
- 파일당 최대 크기는 10MB다.
- 채팅에서는 한 번에 최대 5개를 선택하고, 편집 화면에서는 최대 5개까지 직접 첨부한다.
- 이미지는 전송 전, 채팅 기록, 편집 화면과 일기 상세에서 실제 이미지로 표시한다.
- 이미지가 아닌 파일은 파일명과 크기를 표시하고 다운로드할 수 있다.
- AI 지원 형식은 이미지, PDF와 TXT다.
- 여러 채팅 메시지에 저장된 첨부물은 초안 생성 시 최종 일기의 첨부물 목록으로 이어지며, AI 초안 생성에는 그중 지원 형식 최대 5개를 사용한다.
- 첨부물 원본은 IndexedDB에 Blob으로 저장한다.

### FR-06 달력과 목록

- 달력은 월 단위로 이동한다.
- 오늘과 현재 선택한 날짜를 구분한다.
- 기록이 있는 날짜에는 기분 아이콘, 저장된 날씨 아이콘과 제목을 표시한다.
- 날씨 모름인 기록은 날씨 아이콘을 표시하지 않는다.
- 기록이 없는 날짜를 한 번 누르면 해당 날짜를 선택한다.
- 기록이 있는 날짜를 누르면 상세 화면을 연다.
- 목록은 전체 일기를 최신 날짜순으로 표시한다.
- 선택한 보기 방식은 브라우저 설정에 저장한다.

### FR-07 기록 검색

검색은 브라우저 내부에서 실행하며 Gemini API를 호출하지 않는다.

검색 대상:

- 일기 날짜
- 제목
- 본문
- 같은 날짜에 저장된 대화 텍스트
- 기분
- 날씨

날짜 검색은 다음 형식을 해석한다.

- YYYY-MM-DD
- YYYY년
- M월
- D일
- YYYY년 M월
- M월 D일
- YYYY년 M월 D일

기분 검색은 따뜻함, 행복, 평온, 편안함, 지침, 피곤함, 울적함, 슬픔, 설렘 등의 미리 정의된 표현을 연결한다. 날씨 검색은 맑음, 흐림, 비, 눈과 날씨 모름 관련 표현을 연결한다.

여러 일반 검색어가 있으면 그중 하나라도 날짜, 제목, 본문 또는 대화에 포함된 일기를 결과로 표시한다. 결과는 최신 날짜순으로 보여준다.

사진 내용은 직접 검색하지 않는다. 사진에 관한 설명이 대화 또는 일기 텍스트에 남아 있는 경우에만 해당 텍스트를 통해 검색될 수 있다.

### FR-08 날씨

- 앱 시작 시 Open-Meteo에서 서울의 현재 기온과 날씨를 조회한다.
- 메인 화면 날씨 버튼을 누르면 브라우저 위치 권한을 요청한다.
- 권한을 허용하면 현재 좌표의 날씨를 ‘현재 위치’로 표시한다.
- 권한을 거부하거나 위치 기능을 사용할 수 없으면 서울 날씨를 유지한다.
- 날씨 코드는 맑음, 흐림, 비, 눈으로 단순화한다.
- 조회한 날씨는 설정에 저장한다.
- 시작 화면 효과는 현재 날씨 자동 반영 또는 사용자의 수동 선택을 따른다.

### FR-09 백업과 복원

- 백업은 설정, 프로필, 일기, 세션, 메시지, 첨부 메타데이터와 원본 파일을 포함한다.
- 결과물은 ZIP 파일로 다운로드한다.
- ZIP에는 `manifest.json`, `data.json`, `attachments/` 파일이 들어간다.
- 복원 시 백업 형식 버전과 필수 JSON 파일, 첨부 파일 존재 여부를 확인한다.
- 복원은 기존 데이터를 병합하지 않고 전체 교체한다.
- 복원 전 기존 일기가 있으면 현재 데이터를 ZIP으로 자동 다운로드한다.
- 모든 저장소 교체는 하나의 IndexedDB 트랜잭션에서 실행한다.

### FR-10 저장 공간 관리

- 앱 시작과 서재 입장 시 브라우저 영구 저장 권한을 요청한다.
- 설정 화면에서 저장 사용량과 브라우저가 제공한 할당량을 표시한다.
- 사용되지 않는 첨부물은 앱 시작과 일기 삭제 시 정리한다.
- 모든 로컬 데이터 삭제는 두 번 확인한다.

### FR-11 위기 표현 처리

- 사용자의 새 메시지에 자살·자해 위험을 나타내는 지정 표현이 포함되면 Gemini 호출보다 안전 안내를 먼저 표시한다.
- 안내에는 자살예방 상담전화 109와 긴급 상황의 112·119를 포함한다.
- 해당 메시지와 안전 안내도 대화 기록에 저장한다.

## 9. AI 동작 명세

### 9.1 일반 대화

AI의 역할은 일상 대화 상대이자 서재지기 모아다.

- 선택한 말투를 사용한다.
- 잡담, 공감, 의견, 정보와 질문을 맥락에 맞게 자유롭게 사용한다.
- 모든 답변을 질문으로 끝내지 않는다.
- 일기 작성을 요청받기 전에는 성급하게 요약하거나 교훈으로 마무리하지 않는다.
- 첨부물은 확인 가능한 내용에 근거해 이야기한다.
- 프로필 이름은 자연스러운 순간에만 사용한다.
- 나이, 성별과 생일은 관련 맥락에서만 참고한다.

### 9.2 초안 작성

- 작성 날짜를 함께 전달한다.
- 저장된 대화와 지원 첨부물을 근거로 사용한다.
- 사용자가 말하지 않은 사건과 감정을 만들지 않는다.
- 1인칭 한국어 일기로 작성한다.
- 억지 교훈, 진단과 과장된 위로를 피한다.
- 대화 말투 설정보다 사용자의 표현과 감정 강도를 우선한다.
- 제목과 본문이 포함된 구조화 JSON을 반환한다.

### 9.3 API 구성

- 요청 경로: `POST /api/chat`
- 모드: `chat`, `draft`
- API 키: `GEMINI_API_KEY`
- 모델 재정의: `GEMINI_MODEL`
- 기본 모델 ID: `gemini-3.5-flash`
- 요청 이력: 최근 최대 30개 메시지

## 10. 데이터 저장 설계

### 10.1 저장 위치

모든 제품 데이터의 정본은 현재 사이트 origin의 IndexedDB 데이터베이스 `worldtree-diary`에 저장한다. 브라우저 창을 닫거나 컴퓨터를 다시 시작해도 같은 브라우저 프로필과 같은 사이트 주소를 사용하면 기록은 유지된다.

다음 상황에서는 기록이 사라지거나 다른 저장소처럼 보일 수 있다.

- 사용자가 사이트 데이터를 삭제한 경우
- 시크릿 모드를 종료한 경우
- 브라우저 프로필을 변경하거나 초기화한 경우
- 접속 주소 또는 포트가 달라진 경우
- 브라우저 저장 정책에 따라 데이터가 정리된 경우
- 기기 저장 장치가 손상된 경우

장기 보관과 기기 이동은 ZIP 백업·복원을 사용한다.

### 10.2 IndexedDB 테이블

#### `settings`

| 필드 | 형식 | 설명 |
|---|---|---|
| key | string | 설정 키, 기본 키 |
| value | unknown | 설정 값 |
| updatedAt | ISO datetime | 수정 시각 |

사용 설정에는 `profile`, `chatTone`, `viewMode`, `weather`, `introWeatherMode`, `lastBackupAt`이 포함된다.

#### `diaries`

| 필드 | 형식 | 설명 |

| title | string | 제목 |
| content | string | 최종 본문 |
| source | ai 또는 manual | 작성 방식 |
| mood | string, 선택 | 기분 |
| weather | object, 선택 | 날씨 스냅샷 |
| attachmentIds | UUID[] | 연결 첨부물 |
| createdAt | ISO datetime | 생성 시각 |
| updatedAt | ISO datetime | 수정 시각 |

#### `sessions`

| 필드 | 형식 | 설명 |
|---|---|---|
| id | UUID | 세션 ID |
| date | YYYY-MM-DD | 작성 대상 날짜 |
| diaryId | UUID, 선택 | 완성된 일기 ID |
| tone | string | 세션 말투 |
| status | string | 세션 상태 |
| createdAt | ISO datetime | 생성 시각 |
| updatedAt | ISO datetime | 수정 시각 |

#### `messages`

| 필드 | 형식 | 설명 |
|---|---|---|
| id | UUID | 메시지 ID |

| role | user, assistant 또는 system | 발화자 |

| 필드 | 형식 | 설명 |

## 11. 개인정보와 보안

- 로컬 프로필과 기록은 별도 데이터베이스 서버에 저장하지 않는다.
- 같은 브라우저 프로필을 사용할 수 있는 사람은 로컬 기록을 열람할 수 있다.
- Gemini 대화 시 메시지, 최근 대화 이력, 입력한 프로필과 지원 첨부물이 Google API로 전송된다.
- 지난 기록 검색은 로컬에서 처리하며 검색을 위해 일기 전체를 Gemini에 보내지 않는다.
- API 키는 브라우저 코드가 아니라 서버 환경 변수에서 읽는다.
- `.env` 파일과 API 키는 Git 저장소에 포함하지 않는다.
- 채팅 화면에 AI 전송 내용이 Google에서 처리된다는 안내를 표시한다.
- ZIP 백업에는 민감한 기록과 첨부물이 암호화되지 않은 상태로 들어가므로 안전한 위치에 보관해야 한다.

## 12. 시각 디자인

시각 테마는 몽환적인 분홍빛 세계수, 따뜻한 종이와 서재, 2D 게임풍 캐릭터를 조합한다.

- 배경: 사용자가 제공한 `worldtree-bg.png`를 시작 화면과 메인 화면에 공통 사용
- 본문 색: 잉크 자주색 `#4A3447`
- 보조 글자색: `#755F70`
- 종이색: `#FFF8F7`
- 깊은 종이색: `#F7E6EB`
- 주요 버튼: 장밋빛 `#D97297`
- 보조 포인트: 보라색 `#976CA1`
- 기본 글꼴: Pretendard, Noto Sans KR 계열
- 제목과 일기 본문: MaruBuri, 바탕 계열
- 메인 기록 패널: 반투명 배경과 블러를 적용해 세계수가 비치도록 표현
- 모아 캐릭터: 잎 아이콘, 눈과 반짝임을 조합한 CSS 캐릭터
- 접근성: 키보드 포커스 윤곽선과 모션 감소 환경 설정 대응
- 반응형: 1000px 이하에서 작업 영역을 재배치하고 760px 이하에서 모바일 레이아웃 적용

## 14. 구현 완료 조건

다음 동작이 현재 제품의 완료 기준이다.

- 로컬 프로필을 만든 뒤 메인 서재에 입장할 수 있다.
- 같은 브라우저에서 새로고침하거나 다시 실행해도 프로필과 일기가 남는다.
- AI와 대화하고 초안을 생성한 뒤 수정해 저장할 수 있다.
- AI 키가 없어도 데모 대화와 단순 초안 흐름을 확인할 수 있다.
- 수동으로 오늘 또는 선택한 과거 날짜의 일기를 작성할 수 있다.
- 첨부 이미지를 채팅, 편집과 상세 화면에서 볼 수 있다.
- 저장된 일기를 달력과 목록에서 다시 열 수 있다.
- 완성된 일기의 저장 대화 텍스트를 읽기 전용으로 볼 수 있다.
- 한국어 날짜, 키워드, 대화, 기분과 날씨로 기록을 검색할 수 있다.
- 현재 날씨를 조회하고 시작 화면 효과를 자동 또는 수동으로 선택할 수 있다.
- 사진과 파일을 포함한 ZIP 백업을 생성하고 전체 복원할 수 있다.
- 저장 공간 상태를 확인하고 모든 로컬 데이터를 삭제할 수 있다.
- Vercel용 Nitro 빌드와 코드 검사가 통과한다.

## 15. 최종 범위 문장

나무의 서재는 외부 계정 없이 로컬 프로필로 입장해 Gemini와 대화하거나 직접 날짜별 일기를 작성하고, 대화·사진·파일·기분·날씨를 현재 브라우저에 함께 저장하며, 달력·목록·로컬 검색과 ZIP 백업·복원을 제공하는 반응형 웹 일기 애플리케이션이다.
